# Rush Medical College — AI Research Agent Demo

An end-to-end demonstration of **agentic coding** applied to medical literature review. A medical student types a clinical research question; an AI agent searches PubMed and OpenAlex, screens articles against inclusion/exclusion criteria, retrieves full text, builds an evidence table, and writes a narrative synthesis — all with real, verifiable citations.

Built for the Rush Medical College board meeting demo (May 2026).

## What this demo proves

- A clinical question can be turned into a structured literature review in a few minutes, without humans hand-curating each step.
- Every PMID and DOI is real — pulled from PubMed's live API, not generated from model memory.
- Free databases (PubMed + OpenAlex) cover the heavy lifting; paid APIs (OpenRouter for the LLM) keep the cost per run under a dollar.
- The same agentic pattern that powers this demo also lets a non-engineer modify the code. See [GETTING_STARTED.md](./GETTING_STARTED.md) Step 5.

## Example output at a glance

- **Topic:** Do GLP-1 receptor agonists reduce the risk of stroke in patients with type 2 diabetes?
- **Articles screened:** 20 (from 166 identified)
- **Articles included:** 8 systematic reviews / meta-analyses
- **Full text retrieved:** 5 (via PubMed Central)

The example artifacts in this folder were generated by an earlier prototype using Claude Code with a PubMed MCP server. The live `research-agent/` app produces the same structure, but inside a polished web UI with real-time streaming progress.

## Disclaimers

- AI-assisted, not AI-authored. Every claim requires expert verification.
- Educational software, not medical advice.
- Source coverage is limited to enabled databases — not a publication-grade systematic review.
- No dual independent human screening. Does not meet PROSPERO / PRISMA / GRADE standards for publication.
- Students must comply with institutional academic integrity policies.
